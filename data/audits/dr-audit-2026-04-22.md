## Executive Summary

TopTalent Jobs draait op een moderne stack (Supabase + Vercel + GitHub) met goede security en monitoring. Echter, **de disaster recovery is vrijwel niet-bestaand**:

- Geen enkele door TopTalent beheerde backup (geen pg_dump, geen Storage sync)
- Volledig afhankelijk van Supabase's ingebouwde daily backups (retentie 7-14 dagen)
- Supabase Storage heeft GEEN backup — documentverlies is permanent
- 10 van 19 environment secrets zijn niet herstelbaar zonder lokale `.env.local`
- Bus factor 1: één persoon heeft alle root-toegang

**Bij een catastrofaal incident vannacht is de verwachte uitkomst: database herstelbaar (24u dataverlies), maar alle documenten (CV's, ID-bewijzen, contracten) permanent verloren.**

## TOP-3 DR-gaps (KRITIEK)

1. **Geen Storage backup** — Supabase biedt geen automatische Storage backups. Alle kandidaat-CV's, ID-bewijzen en contracten staan op één plek zonder kopie. Bij verlies: WID-overtreding, SNA-audit faalt, contracten onbewijsbaar.

2. **Geen off-site database backup** — Er wordt volledig vertrouwd op Supabase's ingebouwde daily backups. Als het Supabase-project verwijderd wordt, zijn ook de backups weg. Geen pg_dump, geen export naar externe opslag.

3. **Bus factor 1 + secrets niet in kluis** — Eén persoon heeft alle root-toegang. 10/19 environment secrets zijn alleen beschikbaar in `.env.local` op de lokale machine. Bij laptop-verlies + Vercel-account-verlies zijn deze secrets permanent weg.

## TOP-3 Quick Wins

1. **Alle secrets naar wachtwoordmanager** — Kopieer vandaag nog alle env vars uit `.env.local` naar 1Password of Bitwarden. Effort: 30 minuten. Impact: voorkomt secrets-verlies.

2. **Storage backup script** — Schrijf een script dat wekelijks alle 4 Supabase buckets naar Cloudflare R2 synct (gratis tot 10GB). Effort: 2-4 uur. Impact: voorkomt permanent documentverlies.

3. **PITR activeren** — Controleer in Supabase dashboard of Point-in-Time Recovery actief is. Zo niet: activeer (Pro plan + addon). Effort: 15 minuten. Impact: RPO van 24 uur naar < 5 minuten.

## TOP-3 Vragen voor jezelf

1. **Weet je welk Supabase plan je hebt?** (Free/Pro/Scale-Up) Dit bepaalt de backup-retentie en PITR-beschikbaarheid.

2. **Heb je ooit een restore getest?** Zo niet: een backup die nooit getest is, is geen backup.

3. **Als jij morgen niet bereikbaar bent — kan iemand anders de site herstellen?** Zo niet: schrijf vandaag een nood-SOP en deel het met een vertrouwd persoon.

## Voorstel: Jaarlijkse DR-drill

### Frequentie
- **Kwartaal:** Secret-rotatie + backup-verificatie
- **Halfjaar:** Database restore-test naar staging
- **Jaarlijks:** Volledig DR-drill (DB + Storage + secrets)

### DR-drill checklist
1. Download laatste pg_dump → restore naar Docker Postgres → verifieer tabellen + RLS
2. Download 10 random Storage objects → verifieer integriteit
3. Maak nieuw Vercel-project → deploy vanuit GitHub → alle env vars handmatig invullen → test
4. Meet RTO per stap → documenteer
5. Update runbooks met geleerde lessen

### Doel
- RTO < 2 uur voor database
- RTO < 4 uur voor volledig systeem
- RPO < 5 minuten (met PITR)
- Storage backup niet ouder dan 7 dagen
