## Executive Summary

De TopTalent applicatie heeft **87 database-tabellen** (75 SQL-migratiebestanden). Alle API-routes gebruiken uitsluitend de `supabaseAdmin` (service-role) client, wat betekent dat **alle beveiliging op applicatie-niveau** plaatsvindt. Er is **geen database-niveau fallback** als een API-route een auth-check mist.

**MEEST KRITIEKE BEVINDING:** De `admin_2fa` tabel heeft een `GRANT SELECT, INSERT, UPDATE TO anon` — **iedereen met de anon key kan TOTP-secrets en backup codes lezen, 2FA uitschakelen, en records wijzigen.** Dit ondermijnt de volledige admin-beveiliging.

**Tweede kritieke bevinding:** Veel tabellen die WEL RLS hebben, gebruiken `FOR ALL USING (true)` policies — waardoor ze effectief OPEN staan voor iedereen met de anon key. Dit betreft o.a. `medewerker_documenten`, `berichten`, `certificeringen`, `bookings`, `offertes`, `referrals`.

## Architectuur-overzicht

### Auth-mechanismen (197 routes totaal)

| Mechanisme | Routes | Sterkte | Rij-filtering |
|-----------|--------|---------|---------------|
| `verifyAdmin` | ~80 | Sterk (JWT + email whitelist + 2FA) | Geen (by design) |
| `verifyKlantSession` | ~25 | Sterk (signed JWT cookie, 7d) | `.eq("klant_id", session.id)` — correct |
| `verifyMedewerkerSession` | ~35 | Sterk (signed JWT cookie, 7d) | `.eq("medewerker_id", session.id)` — correct |
| `CRON_SECRET` | ~25 | Sterk (Bearer token) | Query-specifiek |
| HMAC-SHA256 | 4 | Sterk | N.v.t. |
| Token-based | ~5 | Goed | Token → record lookup |
| reCAPTCHA + rate limit | ~5 | Goed (anti-spam) | N.v.t. (write-only) |
| **Geen auth** | **~20** | **Risico verschilt** | **Geen** |

### Positieve bevindingen

1. **Klant- en medewerker-routes** hebben correcte rij-filtering (geen IDOR gevonden in code-analyse)
2. **Wachtwoord-beveiliging** is goed (bcrypt 10 rounds, strength validation)
3. **JWT-sessies** zijn correct geïmplementeerd (HttpOnly, Secure, SameSite)
4. **2FA** is beschikbaar voor admins (TOTP + backup codes)
5. **Webhook-verificatie** is correct (HMAC-SHA256, timing-safe)
6. **Rate limiting** is aanwezig op kritieke routes (Redis-based)

## Top-10 Aanbevolen Fixes

### 1. REVOKE `admin_2fa` anon toegang — VANDAAG (C-6)
```sql
-- ONMIDDELLIJK UITVOEREN IN SUPABASE DASHBOARD → SQL EDITOR
REVOKE SELECT, INSERT, UPDATE ON admin_2fa FROM anon;
REVOKE SELECT, INSERT, UPDATE ON admin_2fa FROM authenticated;

-- Alleen service_role mag erbij
ALTER TABLE admin_2fa ENABLE ROW LEVEL SECURITY;
CREATE POLICY admin_2fa_service_only ON admin_2fa
  FOR ALL USING (current_setting('role') = 'service_role');
```

### 2. Fix ALLE `USING (true)` policies (C-7)
```sql
-- Voor ELKE tabel met USING (true) policies:
-- Stap 1: Drop de misleidende policy
DROP POLICY IF EXISTS "policy_naam" ON tabel_naam;

-- Stap 2: Maak een echte service-role policy
CREATE POLICY service_role_only ON tabel_naam
  FOR ALL USING (current_setting('role') = 'service_role');

-- Herhaal voor: medewerker_werkervaring, medewerker_vaardigheden,
-- medewerker_documenten, berichten, dienst_aanbiedingen,
-- certificeringen, spoeddienst_responses, bookings, referrals,
-- offertes, pricing_rules, google_reviews, content_posts,
-- factuur_regels, bericht_templates, push_subscriptions
```

### 3. ENABLE RLS op tabellen zonder RLS (C-1)
```sql
-- Voor elke tabel zonder RLS:
ALTER TABLE medewerkers ENABLE ROW LEVEL SECURITY;
CREATE POLICY service_role_only ON medewerkers
  FOR ALL USING (current_setting('role') = 'service_role');

-- Herhaal voor: klanten, inschrijvingen, kandidaat_documenten,
-- linkedin_connections, chatbot_conversations, chatbot_messages,
-- email_log, audit_log, acquisitie_leads, calculator_leads,
-- admin_2fa + alle overige tabellen zonder RLS
```

### 4. Fix storage policies (C-8)
```sql
-- Drop te brede policy
DROP POLICY IF EXISTS "Admin can upload kandidaat documenten" ON storage.objects;

-- Maak restrictieve policy (alleen service_role)
CREATE POLICY "Service role kandidaat documenten"
ON storage.objects FOR ALL
TO service_role
USING (bucket_id = 'kandidaat-documenten')
WITH CHECK (bucket_id = 'kandidaat-documenten');
```
